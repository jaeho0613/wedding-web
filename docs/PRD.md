# PRD: 모바일 웨딩 청첩장

**작성일**: 2026-04-04  
**버전**: 1.0  
**상태**: 초안

---

## 1. 프로젝트 개요

### 목적
결혼식 초대장 및 정보를 하객들과 공유하기 위한 모바일 최적화 웹 페이지. Notion을 CMS로 활용하여 콘텐츠를 별도 개발 없이 관리할 수 있도록 한다.

### 핵심 가치
- **쉬운 콘텐츠 관리**: Notion에서 수정하면 웹 페이지에 자동 반영
- **모바일 최적화**: 하객 대부분이 모바일로 접근하는 상황에 맞춘 UI
- **단순함**: 하객이 정보를 빠르게 확인할 수 있는 직관적인 구성

## 2. 사용자 스토리

| 우선순위 | 사용자 | 목표 | 이유 |
|---------|--------|------|------|
| MVP | 하객 | 결혼식 일시·장소를 확인한다 | 참석 여부와 이동 계획을 세우기 위해 |
| MVP | 하객 | 신랑·신부 연락처를 확인한다 | 필요 시 연락하기 위해 |

| MVP | 신랑·신부 | Notion에서 정보를 수정하면 웹에 자동 반영된다 | 개발자 없이 콘텐츠를 직접 관리하기 위해 |

## 3. 기능 요구사항

### 3.1 Notion 데이터 연동

**승인 기준**
- 주어진: Notion 데이터베이스에 결혼식 정보가 저장되어 있을 때
- 할 때: 하객이 웹 페이지에 접속하면
- 그러면: Notion의 최신 데이터가 웹 페이지에 표시된다

**세부 요구사항**
- Notion API는 Vercel Edge Functions를 통해 호출 (API 키 클라이언트 노출 방지)
- 데이터 캐싱: ISR(Incremental Static Regeneration) 또는 Edge Functions의 캐시 활용
- Notion API 오류 시 fallback 처리

### 3.2 뷔폐 메뉴 표시

**승인 기준**
- 주어진: Notion 데이터베이스에 뷔폐 메뉴 항목이 있을 때
- 할 때: 하객이 뷔폐 정보 섹션으로 스크롤하면
- 그러면: 메뉴 목록이 카드 또는 리스트 형태로 표시된다

**세부 요구사항**
- 메뉴명, 카테고리(한식/양식/디저트 등) 표시
- Notion에서 메뉴 추가/수정/삭제 시 자동 반영

### 3.3 반응형 디자인

**승인 기준**
- 주어진: 모바일(375px~), 태블릿(768px~), 데스크탑(1280px~) 뷰포트에서
- 할 때: 페이지에 접속하면
- 그러면: 각 화면 크기에 맞는 레이아웃으로 표시된다

**세부 요구사항**
- 모바일 우선(mobile-first) 설계
- Tailwind CSS v4 반응형 유틸리티 사용

## 4. 화면 구성

### 섹션 순서 (위 → 아래)

```
┌─────────────────────────────┐
│  1. 청첩장 메인 섹션          │  결혼식 날짜, 장소, 커플 이름
├─────────────────────────────┤
│  2. 소개글 섹션               │  두 사람의 짧은 인사말
├─────────────────────────────┤
│  3. 신랑·신부 연락처 섹션     │  전화번호, SNS 등
├─────────────────────────────┤
│  4. 웨딩 사진 갤러리 섹션     │  스와이프 또는 그리드 갤러리
├─────────────────────────────┤
│  5. 뷔폐 정보 섹션            │  메뉴 목록 카드
└─────────────────────────────┘
```

### 섹션별 상세

#### 1. 청첩장 메인 섹션
- 배경 이미지 또는 일러스트
- 신랑·신부 이름 (한문/영문)
- 결혼식 날짜, 시간
- 결혼식장 이름 및 주소

#### 2. 소개글 섹션
- 신랑·신부 또는 양가의 짧은 메시지
- Notion 본문 텍스트로 관리

#### 3. 신랑·신부 연락처 섹션
- 신랑 정보: 이름, 전화번호
- 신부 정보: 이름, 전화번호
- 전화 걸기 버튼 (tel: 링크)

#### 4. 웨딩 사진 갤러리 섹션
- Notion에서 관리되는 이미지 URL 목록
- 모바일: 스와이프 캐러셀
- 데스크탑: 그리드 레이아웃

#### 5. 뷔폐 정보 섹션
- Notion 데이터베이스에서 가져온 메뉴 항목
- 카테고리별 그룹핑

## 5. 기술 상세

### 5.1 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| Frontend | React.js 19, TypeScript | UI 구성 |
| Styling | Tailwind CSS v4, Shadcn/ui | 스타일링 및 컴포넌트 |
| Icons | lucide-react | 아이콘 |
| CMS | Notion API (@notionhq/client) | 콘텐츠 관리 |
| Deployment | Vercel | 배포 및 Edge Functions |

### 5.2 Notion 데이터베이스 스키마

#### 결혼식 정보 (단일 페이지)
```
결혼식 정보
├── 신랑 이름 (Title)
├── 신부 이름 (Text)
├── 결혼식 날짜 (Date)
├── 결혼식 시간 (Text)
├── 결혼식장 이름 (Text)
├── 결혼식장 주소 (Text)
├── 신랑 전화번호 (Text)
├── 신부 전화번호 (Text)
├── 소개글 (Text / 본문)
└── 갤러리 이미지 URL (Files & Media 또는 Text)
```

#### 뷔폐 메뉴 (데이터베이스)
```
뷔폐 메뉴
├── 메뉴명 (Title)
├── 카테고리 (Select: 한식 | 양식 | 일식 | 디저트 | 음료)
├── 설명 (Text, optional)
└── 순서 (Number, 정렬용)
```

### 5.3 Vercel Edge Functions API 엔드포인트

```
GET /api/wedding-info
  → Notion 결혼식 정보 페이지 데이터 반환

GET /api/buffet-menu
  → Notion 뷔폐 메뉴 데이터베이스 항목 반환
```

**응답 예시 (`/api/wedding-info`)**
```json

  "brideName": "김영희",
  "date": "2026-06-07",

  "venueAddress": "서울시 강남구 테헤란로 123",
  "groomPhone": "010-1234-5678",
  "bridePhone": "010-8765-4321",
  "introduction": "저희 두 사람이 하나가 됩니다...",

```

## 6. 비기능 요구사항

### 성능
- 초기 로드: 3G 기준 3초 이내 콘텐츠 표시 (LCP)
- 이미지: 지연 로딩(lazy loading) 적용
- Notion API 응답 캐싱으로 불필요한 API 호출 최소화

### 보안
- Notion API 키는 Vercel 환경 변수로 관리 (클라이언트에 노출 금지)
- Edge Functions에서만 Notion API 직접 호출

### 접근성
- 모바일 터치 친화적 UI (버튼 최소 44px)
- 이미지 alt 텍스트 제공
