개발 순서:
## Phase 1: 프로젝트 초기 설정 (1-2일)
- React.js 프로젝트 구조 설정
- Notion API 연동 환경 구축
- 기본 레이아웃 구조 생성
- 이유: 견고한 기반 없이는 기능 개발이 어려움

## Phase 2: 공통 모듈 개발 (2-3일)
- Notion API 공통 함수 (fetchPages, fetchPageContent)
- 공통 컴포넌트 (Header, Footer, Section)
- 공통 타입 정의
- 이유: 모든 기능에서 재사용되는 코드를 먼저 만들어야 중복 방지

## Phase 3: 핵심 기능 개발 (3-4일)
- 각 세션 별 컴포넌트 제작
- Notion 컨텐츠 렌더링
- 이유: 블로그의 가장 기본이 되는 기능

## Phase 4: 최적화 및 배포 (1-2일)
- 성능 최적화
- 반응형 디자인 개선
- Vercel 배포
- 이유: 기능이 완성된 후 품질 향상

각 Phase마다 예상 소요 시간과 완료 기준을 포함해서 작성해줘.
